# Common Chapter 2: 프로그래밍 언어와 컴파일/링킹

---

## 오늘 목표
- 프로그래밍 언어의 종류를 구분한다
- 컴파일러/인터프리터 차이를 이해한다
- 오브젝트 코드와 링킹 과정을 설명한다
- 심볼과 심볼 테이블의 역할을 이해한다

---

## 프로그래밍 언어란?
- 사람이 읽고 쓰는 형태의 규칙(문법)
- 컴퓨터가 실행 가능한 코드로 변환 필요

---

## 언어 분류
- 컴파일 언어: C, C++
- 인터프리터 언어: Python, JavaScript
- 혼합형: Java, C# (바이트코드 + VM)

---

## 컴파일러 vs 인터프리터
- 컴파일러: 소스 → 실행 파일 생성
- 인터프리터: 실행 시점에 해석/실행

---

## 컴파일 과정 (개요)
1. 전처리(Preprocess)
2. 컴파일(Compile)
3. 어셈블(Assemble)
4. 링킹(Link)

---

## 오브젝트 코드
- 소스 파일 하나를 컴파일한 결과
- 기계어 + 심볼 정보 포함
- 아직 실행 파일 아님

---

## 링커(Linker)의 역할
- 여러 오브젝트 파일을 하나로 결합
- 외부 라이브러리 연결
- 실행 파일 생성

---

## 심볼(Symbol)
- 함수/변수 이름 같은 식별자
- 링커가 참조 위치를 찾아 연결

---

## 심볼 테이블
- 심볼 이름과 주소/속성 정보의 목록
- 링킹 시 누락/중복을 검사

---

## 링킹 에러 예시
- Undefined reference: 선언만 있고 정의 없음
- Multiple definition: 동일 심볼이 여러 번 정의

---

## 정적 vs 동적 링크
- 정적 링크: 라이브러리 코드 포함
- 동적 링크: 실행 시 라이브러리 로드

---

## 실행 파일의 구성(개념)
- 코드 영역(text)
- 데이터 영역(data)
- 스택/힙

---

## 요약
- 언어는 컴파일/해석 과정을 거친다
- 링커가 심볼을 연결해 실행 파일을 만든다
- 심볼 테이블은 링크의 핵심 자료구조
