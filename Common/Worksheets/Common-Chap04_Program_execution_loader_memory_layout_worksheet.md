# Common 워크시트 Chapter 4: 실행 파일과 메모리 레이아웃

---

## 학습 목표
- 실행 파일이 프로세스로 전환되는 과정을 단계별로 설명한다
- 메모리 구역(Text/Data/BSS/Heap/Stack)을 구분한다
- 함수 호출과 스택 프레임의 관계를 관찰한다

---

## 실습 1: 메모리 구역별 변수 선언
요구사항:
- 전역 변수(초기화 O/X), 지역 변수, `malloc` 변수 선언
- 각 변수 주소를 출력

힌트:
```c
printf("%p\n", (void *)&global_var);
```

---

## 실습 2: 함수 호출 스택 관찰
요구사항:
- `main -> a -> b` 순으로 함수 호출
- 각 함수에서 지역 변수 주소 출력
- 호출 깊이에 따라 주소가 어떻게 변하는지 기록

---

## 실습 3: 재귀 깊이 관찰
요구사항:
- 재귀 함수에서 depth와 지역 변수 주소 출력
- depth 증가 시 주소 패턴을 관찰

체크:
- [ ] 스택이 "호출마다 추가"된다는 점을 설명할 수 있는가?

---

## 개념 질문
1. 로더가 실행 직전에 하는 작업 3가지를 적으세요.
2. Text/Data/BSS/Heap/Stack의 용도를 각각 한 줄로 적으세요.
3. Stack Overflow가 발생하는 대표 상황을 적으세요.
